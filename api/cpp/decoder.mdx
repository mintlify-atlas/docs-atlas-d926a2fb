## Overview

The `zbar::Decoder` class provides a low-level interface for decoding barcodes from a stream of bar and space widths. It identifies symbols and extracts encoded data without requiring image data.

**Header:** `<zbar.h>` (includes `zbar/Decoder.h`)

**Namespace:** `zbar`

## Handler Class

### Decoder::Handler

Abstract base class for handling decode results.

```cpp
class Handler {
public:
    virtual ~Handler();
    virtual void decode_callback(Decoder &decoder) = 0;
};
```

Applications should subclass this and implement `decode_callback()` to process decode results.

**Example:**
```cpp
class MyHandler : public Decoder::Handler {
    void decode_callback(Decoder &decoder) {
        cout << "Decoded: " << decoder.get_data() << endl;
    }
};
```

## Decoding Methods

### decode_width()

```cpp
zbar_symbol_type_t decode_width(unsigned width)
```

Processes the next bar or space width from the input stream.

<ParamField path="width" type="unsigned" required>
  Width of the next bar (if current color is black) or space (if white)
</ParamField>

<ResponseField name="return" type="zbar_symbol_type_t">
  The symbol type if a symbol was just decoded, otherwise `ZBAR_NONE` (0) or `ZBAR_PARTIAL`
</ResponseField>

**Example:**
```cpp
zbar_symbol_type_t type = decoder.decode_width(42);
if (type != ZBAR_NONE && type != ZBAR_PARTIAL) {
    cout << "Decoded: " << decoder.get_data() << endl;
}
```

### operator&lt;&lt;()

```cpp
Decoder& operator<<(unsigned width)
```

Convenience operator for processing bar/space widths.

<ParamField path="width" type="unsigned" required>
  Width to process
</ParamField>

<ResponseField name="return" type="Decoder&">
  Reference to this decoder for chaining
</ResponseField>

**Example:**
```cpp
decoder << 10 << 20 << 15 << 25;
```

## State Query Methods

### get_color()

```cpp
zbar_color_t get_color() const
```

Retrieves the color of the **next** element to be passed to the decoder.

<ResponseField name="return" type="zbar_color_t">
  `ZBAR_SPACE` (white/background) or `ZBAR_BAR` (black/foreground)
</ResponseField>

**Example:**
```cpp
if (decoder.get_color() == ZBAR_BAR) {
    // Next width should be a bar (black)
}
```

### get_type()


<ResponseField name="return" type="zbar_symbol_type_t">

## Handler Setup

### set_handler()

```cpp
void set_handler(Handler &handler)
```

Sets up a callback handler to process decode results.

<ParamField path="handler" type="Handler&" required>
  Reference to handler object implementing `decode_callback()`
</ParamField>

**Example:**
```cpp
class MyHandler : public Decoder::Handler {
public:
    void decode_callback(Decoder &decoder) {
        cout << "Symbol: " << decoder.get_symbol_name() << endl;
        cout << "Data: " << decoder.get_data() << endl;
        cout << "Direction: " << decoder.get_direction() << endl;
    }
};

Decoder decoder;
MyHandler handler;
decoder.set_handler(handler);
```

## Complete Example

```cpp
#include <iostream>
#include <vector>
#include <zbar.h>

using namespace std;
using namespace zbar;

// Custom handler for decode results
class MyDecodeHandler : public Decoder::Handler {
public:
    void decode_callback(Decoder &decoder) {
        cout << "========== Decode Result =========" << endl;
        cout << "Type:      " << decoder.get_symbol_name() << endl;
        cout << "Data:      " << decoder.get_data() << endl;
        cout << "Length:    " << decoder.get_data_length() << endl;
        cout << "Direction: " << decoder.get_direction() << endl;
        cout << "==================================" << endl;
    }
};

int main() {
    // Create decoder
    Decoder decoder;
    
    // Configure decoder
    decoder.set_config(ZBAR_NONE, ZBAR_CFG_ENABLE, 1);
    
    // Set up handler
    MyDecodeHandler handler;
    decoder.set_handler(handler);
    
    // Example: Decode from a stream of bar/space widths
    // These widths would typically come from a Scanner
    vector<unsigned> widths = {
        10, 20, 10, 30, 15, 25, 20, 10,
        15, 20, 25, 15, 10, 30, 20, 15
        // ... more widths
    };
    
    // Start a new scan
    decoder.new_scan();
    
    // Process each width
    for (unsigned width : widths) {
        zbar_symbol_type_t result = decoder.decode_width(width);
        
        // Check if a symbol was decoded
        if (result != ZBAR_NONE && result != ZBAR_PARTIAL) {
            // Handler will be called automatically
            // But we can also access data directly:
            cout << "Direct access: " << decoder.get_data() << endl;
        }
    }
    
    return 0;
}
```

## Usage with Scanner

The Decoder is typically used together with a Scanner, which converts intensity values to bar/space widths:

```cpp
#include <zbar.h>

using namespace zbar;

// Create decoder and scanner
Decoder decoder;
Scanner scanner(&decoder);  // Scanner feeds decoder automatically

// Configure
decoder.set_config(ZBAR_NONE, ZBAR_CFG_ENABLE, 1);

// Set up handler
MyDecodeHandler handler;
decoder.set_handler(handler);

// Start scanning
scanner.new_scan();

// Feed intensity values (0-255) to scanner
for (int y : intensity_values) {
    scanner.scan_y(y);
    // Scanner automatically calls decoder.decode_width()
    // Handler is invoked when symbols are decoded
}
```

## Symbology Types

Common `zbar_symbol_type_t` values:
- `ZBAR_NONE` - No symbol or used to configure all types
- `ZBAR_EAN8` - EAN-8
- `ZBAR_EAN13` - EAN-13
- `ZBAR_UPCA` - UPC-A
- `ZBAR_UPCE` - UPC-E
- `ZBAR_ISBN10` - ISBN-10
- `ZBAR_ISBN13` - ISBN-13
- `ZBAR_CODE39` - Code 39
- `ZBAR_CODE93` - Code 93
- `ZBAR_CODE128` - Code 128
- `ZBAR_CODABAR` - Codabar
- `ZBAR_I25` - Interleaved 2 of 5
- `ZBAR_DATABAR` - GS1 DataBar
- `ZBAR_DATABAR_EXP` - GS1 DataBar Expanded
- `ZBAR_QRCODE` - QR Code
- `ZBAR_SQCODE` - SQ Code

## Configuration Options

Common `zbar_config_t` values:
- `ZBAR_CFG_ENABLE` - Enable/disable symbology (0 or 1)
- `ZBAR_CFG_MIN_LEN` - Minimum data length
- `ZBAR_CFG_MAX_LEN` - Maximum data length
- `ZBAR_CFG_UNCERTAINTY` - Tolerance for widths
- `ZBAR_CFG_POSITION` - Enable position tracking
