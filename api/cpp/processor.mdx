## Overview

The `zbar::Processor` class provides a high-level, self-contained interface for processing video streams and images for barcodes. It manages video capture, barcode scanning, and optional display output in a single integrated component.

**Header:** `<zbar.h>` (includes `zbar/Processor.h`)

**Namespace:** `zbar`

## Constants

### FOREVER

```cpp
static const int FOREVER = -1;
```

Constant value to pass for timeout parameters to wait indefinitely.

## Constructor

### Processor()

```cpp
Processor(bool threaded = true, 
          const char *video_device = "", 
          bool enable_display = true)
```

Creates and initializes a new Processor.

<ParamField path="threaded" type="bool" default="true">
  If true, runs processing in a separate thread
</ParamField>

<ParamField path="video_device" type="const char*" default='""'>
  Path to video device (e.g., "/dev/video0"). Empty string for no device.
</ParamField>

<ParamField path="enable_display" type="bool" default="true">
  If true, enables the display window
</ParamField>

**Throws:** `std::bad_alloc` if memory allocation fails

**Example:**
```cpp
// Create processor with video device
Processor proc(true, "/dev/video0", true);

// Create processor without initial device
Processor proc(true);
```

### set_config() (integer)

**Example:**

proc.set_config(ZBAR_QRCODE, ZBAR_CFG_MIN_LEN, 10);

## Processing Methods

### process_one()

```cpp
void process_one(int timeout = FOREVER)
```

Processes video frames until a barcode is decoded or timeout occurs.

<ParamField path="timeout" type="int" default="FOREVER">
  Timeout in milliseconds, or `FOREVER` (-1) to wait indefinitely
</ParamField>

**Throws:** Exception on error

**Example:**
```cpp
// Wait indefinitely for a barcode
proc.process_one();

// Wait up to 5 seconds
proc.process_one(5000);
```

### process_image()

```cpp
void process_image(Image &image)
```

Processes a single image for barcodes and optionally displays it.

<ParamField path="image" type="Image&" required>
  The image to process
</ParamField>

**Throws:** Exception on error

**Example:**
```cpp
Image image(640, 480, "Y800", data, size);
proc.process_image(image);
```

## User Interaction

### user_wait()

```cpp
int user_wait(int timeout = FOREVER)
```

Waits for user input to the display window (keyboard or mouse events).

<ParamField path="timeout" type="int" default="FOREVER">
  Timeout in milliseconds, or `FOREVER` (-1) to wait indefinitely
</ParamField>

<ResponseField name="return" type="int">
  1 if input received, 0 on timeout, -1 on error
</ResponseField>

**Throws:** Exception on error

**Example:**
```cpp
try {
    proc.set_visible();
    proc.set_active();
    proc.user_wait();  // Wait for user to close window
} catch (ClosedError &e) {
    // Window was closed
}
```

## Complete Example

```cpp
#include <iostream>
#include <zbar.h>

using namespace std;
using namespace zbar;

// Custom handler for processing results
class MyHandler : public Image::Handler {
public:
    void image_callback(Image &image) {
        for (SymbolIterator symbol = image.symbol_begin();
             symbol != image.symbol_end(); ++symbol) {
            cout << "Decoded " << symbol->get_type_name() 
                 << " symbol: " << symbol->get_data() << endl;
        }
    }
};

int main(int argc, char **argv) {
    // Get video device from command line or use default
    const char *device = (argc > 1) ? argv[1] : "/dev/video0";

    try {
        // Create and initialize processor
        Processor proc(true, device, true);

        // Configure barcode detection
        proc.set_config(ZBAR_NONE, ZBAR_CFG_ENABLE, 1);

        // Set up result handler
        MyHandler handler;
        proc.set_handler(handler);

        // Show preview window and start processing
        proc.set_visible();
        proc.set_active();

        cout << "Press any key in the window to exit..." << endl;

        // Wait for user input
        proc.user_wait();

    } catch (ClosedError &e) {
        cout << "Window closed" << endl;
    } catch (exception &e) {
        cerr << "Error: " << e.what() << endl;
        return 1;
    }

    return 0;
}
```

## Threading Behavior

When the Processor is created with `threaded = true`:
- Video capture and processing run in a separate thread
- The display window can be updated asynchronously
- Result handlers are called from the processing thread
- Thread-safe access to results is managed internally

When `threaded = false`:
- All operations run synchronously on the calling thread
- Methods like `process_one()` block until completion
- Simpler for single-threaded applications
